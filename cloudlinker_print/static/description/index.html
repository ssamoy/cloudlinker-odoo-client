CloudLinker Print

Odoo 18 Printing Automation

CloudLinker Print connects your Odoo instance to the CloudLinker cloud-to-local print service. Trigger print jobs on any locally connected printer — directly from Odoo — without VPNs, without exposing your network, and without installing print drivers on the server.

What is CloudLinker?

CloudLinker is a lightweight SaaS platform that bridges cloud applications and local devices. A small agent runs on a Windows or Linux machine, registers its connected printers, and polls CloudLinker for pending jobs. When a job arrives, it downloads the PDF from the provided URL and sends it to the target printer — all without requiring any inbound firewall ports.

How It Works

Configure

Login or register from Settings → CloudLinker, or enter your Organization ID and API key manually.

Install Agent

Download the CloudLinker client for Windows or Linux from the settings page and install it on your print machine.

Sync Printers

Click Sync Printers to pull all clients and devices from your CloudLinker account.

Set Rules

Go to CloudLinker → Document Rules and assign a default printer per document type.

Print

Use the Print button on any invoice, sale order, picking, or POS order.

Supported Document Types

  • Invoices & Credit Notes — triggered from the invoice form view
  • Sale Orders & Quotations — triggered from the sale order form view
  • Picking / Warehouse Labels — triggered from stock.picking
  • Shipping Labels — triggered from stock.picking (separate rule)
  • Custom QWeb Reports — any ir.actions.report can be linked to a rule
  • POS Receipts — triggered from pos.order

Configuration

  1. In Odoo, go to Settings → CloudLinker and click Login or Create Account to connect your CloudLinker account. You can also enter your Organization ID and API Key manually.
  2. Click Test Connection to verify.
  3. Download and install the CloudLinker client on each machine with a local printer (available from the settings page or from cloudlinker.eu/downloads).
  4. Click Sync Printers to import your clients and printers.
  5. Go to CloudLinker → Document Rules and configure the default printer for each document type.
Note: The document URL sent to the CloudLinker agent must be reachable from the machine running the agent. If your Odoo server is on an internal network, make sure the web.base.url system parameter points to a URL accessible from those machines.

Technical Details

  • Uses the CloudLinker REST API with HTTP Basic Authentication
  • PDF is rendered by Odoo and temporarily stored as an ir.attachment
  • A secure one-time token URL is generated for the CloudLinker agent to fetch
  • No base64 encoding — CloudLinker fetches the PDF directly via URL
  • Compatible with Odoo 18 Community & Enterprise
